# 安装与部署

我们提供灵活的部署选项以满足不同的需求，从本地开发到大规模生产环境。

## 部署方式

- **[Docker Compose 部署](./docker-compose.md) (推荐)**
  
  适用于 **本地开发**、**测试** 和 **轻量级生产** 环境。
  - 单条命令即可完成部署。
  - 包含所有必要组件（网关、认证、市场、Web 等）。
  - 易于配置和维护。

- **[Helm Chart 部署](./helm-deploy.md)**
  
  专为 **Kubernetes** 环境和 **大规模生产** 设计。
  - 高可用性和可扩展性。
  - 云原生集成。
  - 通过 Helm 管理，易于升级和回滚。

## 我该选择哪种方式？

| 特性 | Docker Compose | Helm Chart |
| :--- | :--- | :--- |
| **适用场景** | 本地, 开发, 测试, 中小型生产 | Kubernetes, 大规模生产 |
| **复杂度** | 低 | 中/高 |
| **前置要求** | Docker, Docker Compose | Kubernetes 集群, Helm |
| **扩展性** | 垂直扩展 (主要) | 水平扩展 (自动伸缩) |
