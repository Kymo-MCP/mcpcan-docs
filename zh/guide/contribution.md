# 成为贡献者

能想要为 MCPCAN 贡献力量，真的太赞了！我们早就盼着能迎来像你这样的伙伴，特别期待看到你的创意和付出～
虽然我们是个不到五人的小团队，但心里藏着大大的目标 —— 要打造出最直观、最好用的 MCP 管理平台。所以社区里每一份帮助，对我们来说都是无比珍贵的支持。<br>
<br>结合目前的团队规模，我们需要快速灵活地推进项目落地，但更希望你在贡献过程中能少走弯路、体验顺畅。<br><br>这份贡献指南就是为你准备的，帮你快速熟悉代码库和我们的协作方式，让你能尽快投入到真正有意思的开发环节中。
和 MCPCAN 这个项目一样，这份指南也在不断迭代完善。如果偶尔发现它和实际项目有出入，还请你多多理解，也非常欢迎你提出任何反馈，帮我们把它做得更好。<br>
<br>最后，麻烦你花一分钟看看我们的[许可证](https://github.com/Kymo-MCP/mcpcan/blob/main/LICENSE)、[贡献者协议](https://github.com/Kymo-MCP/mcpcan/blob/main/CONTRIBUTING.md)，还有[社区行为准则](https://github.com/Kymo-MCP/mcpcan/blob/main/CODE_OF_CONDUCT.md)～ 这些是我们一起维护良好协作环境的基础呀！


## 在开始之前
我们对这两种情况都有相应的解决方案。<br>
1、[查看](https://github.com/Kymo-MCP/mcpcan/issues)原有的问题<br>
2、[创建](https://github.com/Kymo-MCP/mcpcan/issues/new/choose)新问题

### 功能请求：
● 如果你想从现有问题中选择一个，请在其下方留下评论表示你的意愿。<br>
● 如果你要提出新的功能请求，请解释所提议的功能的目标，并尽可能提供详细的上下文。我们给您设定了预制的请求模版，你可以参照模版的框架直接填写，我们会根据优先级进行功能请求的优化。下面是我们提供的三个模版。<br>

优先级的评判标准:
| 要素类型 | 优先事项 |
|----------|----------|
| 由团队成员标记的高优先级功能 | 高优先级 |
| 来自我们社区反馈板的热门功能请求 | 中等优先级 |
| 非核心功能和细微改进 | 低优先级 |
| 有价值但并非立竿见影 | 未来功能 |
<br>

### 其他任何事情（例如 bug 报告、性能优化、拼写错误更正）：
● 立即开始编码<br>
<br>优先级的评判标准：
| 问题类型 | 优先事项 |
|----------|----------|
| 核心功能存在漏洞（无法登录、应用程序无法运行、安全漏洞） | 高优先级 |
| 修复非关键性错误，提升性能 | 中等优先级 |
| 小幅修复（拼写错误、界面混乱但功能正常） | 低优先级 |
