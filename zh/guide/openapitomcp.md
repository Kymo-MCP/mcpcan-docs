# OpenAPI文档封装MCP服务案例


## 1. 案例概述

本案例旨在详细说明从实例管理页面出发，选择导入OpenAPI创建实例的完整操作流程，重点涵盖导入OpenAPI模式下基本信息的填写规范，为用户提供清晰的操作指引。

## 2. 前置条件

- 用户已成功登录系统，具备访问实例管理页面的权限；

- 已明确实例创建所需的基础信息（如名称、访问模式、MCP协议相关配置等）；

- MCP服务器已完成部署并可正常访问。

## 3. 操作步骤

### 3.1 进入实例创建页面

1. 登录系统后，在左侧导航栏中找到并点击【实例管理】，进入实例管理页面；
![alt text](../../public/images/image_3.1.1.png)

2. 在实例管理页面的右上角，点击【导入OpenAPI】按钮，弹出实例创建选项表单。
![alt text](../../public/images/image_apitomcp_3.1.2.png)

### 3.2 选择导入OpenAPI创建模式

1. 在弹出的实例创建选项弹窗中，显示三种创建模式（具体以实际系统为准），选择【导入OpenAPI】；

2. 点击【导入OpenAPI】选项，进入导入OpenAPI实例创建的表单。

### 3.3 填写OpenAPI文档基本信息
![alt text](../../public/images/image_apitomcp_3.3.png)

进入表单填写页面后，照页面提示依次完成以下的信息进行填写：

1. **名称**：输入实例的名称，名称需符合系统命名规范（如长度限制、特殊字符限制等，具体以实际系统要求为准），建议名称清晰易懂，能准确标识实例用途；

2. **环境选择**：点击下拉框，从可选的容器环境中选择

3. **服务地址**：根据实际业务需求，填写对应的服务地址

4. **描述**：输入实例的相关描述信息，简要说明实例的用途、适用场景等，便于后续实例的管理和维护。
   
5. **上传OpenAPI文档**：上传接口文档，并仅支持OpenAPI 3.0.0以上的版本的YAML或是JSON格式文件。
